# 容易混淆的点

### `#{diff}` vs `@{get_changed_files}`

```text
#{diff}
```

是把当前 git diff 作为上下文直接发给模型。

```text
@{get_changed_files}
```

是给模型一个工具，让它自己获取当前 git diff。

简单理解：

```text
#{diff}              我把 diff 给你看
@{get_changed_files} 你自己去看 diff
```

### `#{terminal}` vs `@{run_command}`

```text
#{terminal}
```

分享已有终端输出。

```text
@{run_command}
```

让模型执行新命令。

简单理解：

```text
#{terminal}      看刚才终端输出
@{run_command}   现在去跑一个命令
```

### `/fetch` vs `@{fetch_webpage}`

```text
/fetch <url>
```

你手动把网页内容加入 chat。

```text
@{fetch_webpage}
```

让 LLM 使用网页读取工具。

简单理解：

```text
/fetch              我手动取网页
@{fetch_webpage}    你需要时自己取网页
```

### `@{files}` vs `@{agent}`

```text
@{files}
```

偏文件操作：搜、读、创建、修改、看 changed files。

```text
@{agent}
```

偏完整 agent：包含 files 能力，还能 run command、delete file、get diagnostics、ask questions。
